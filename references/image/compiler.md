# gpt-image-2 프롬프트 컴파일러

거칠고 모호한 이미지 요청을 이미지 생성 러너(runner)에 넘길 완성 한국어 프로덕션 프롬프트로 바꾼다. 이미지를 직접 생성하지 않는다. 생성·양산 실행은 러너 소관.

## 1. 워크플로우

| 단계 | 실행 | 판정 문장 |
|---|---|---|
| 1 | 거친 요청에서 빠진 카테고리·컷타입·피사체·스타일·구도·텍스트·AR을 추론해 채운다. 취향 디테일은 질문하지 않고 결정한다. 질문 대상은 정확한 한글 문구·브랜드명·민감 소재뿐이다. | 빈칸 때문에 멈추면 틀렸고, 결과를 바꾸는 하드라인만 질문하면 맞다. |

## 2. 철칙 — 절대 어기지 말 것

1. **앞머리 `[AR x:y SIZE wxh]` 브래킷 금지.** size는 실행 파라미터로만 둔다. 프롬프트엔 **끝에 `AR x:y` 토큰 하나만** 둔다. 슬롯 토큰 `[PERSONA_LOCK]`류는 **작성 단계 전용**이다. 최종 프롬프트에 브래킷이 잔존하면 실격이다.
2. **네거티브 기본 금지 + 화이트리스트 2종.** gpt-image-2는 장면 네거티브를 렌더하는 경향이 있다. **장면 배제는 전부 긍정형**으로 쓴다. 군중 배제는 "프레임 안엔 인물 한 명, 단독", 배경 배제는 "깨끗한 단색 배경", 워터마크 배제는 "브랜드 없는 클린 마감"으로 재서술한다. 예외는 아래 두 레인뿐이며, 이는 우회가 아니라 **컴플라이언스 스티어링**이다.

   | 티어 | 조건 | 허용 문구 |
   |---|---|---|
   | **Tier-0 기본** | 항상 | all-positive. 부정문 0개 |
   | **Tier-1 텍스트 렌더 가드** | 렌더 텍스트(따옴표 카피)가 실제로 있을 때만 | 화이트리스트 7종: `no extra words` · `no duplicate text` · `no invented glyphs` · `no watermark` · `no logo` · `no extra text` · `verbatim, no extra characters` |
   | **Tier-2 화보 컴플라이언스 레인** | 명시 선언 시만. 휴리스틱 승격 금지 | SAFETY_ASSERT(긍정형, 피사체절) + NEGATIVE_TAIL(AR 직전 정확히 1회) **페어** — 고정 문자열, 상세는 화보 레퍼런스 |

   Tier-1 결합 공식(문서가 권장하는 유일한 방출형 1줄): `All text appears once, perfectly legible — no duplicate text, no extra words, no invented glyphs, no watermark.`

   Tier-2 고정 문자열 — SAFETY_ASSERT: `adult Korean woman in her late 20s, 25+, original character, non-nude fashion editorial styling, fully opaque fabric, covered chest line, editorial upright pose` / NEGATIVE_TAIL: `no nudity, no nipple or genital exposure, no wardrobe malfunction, no extra people, no text, no watermark` (순서 보존 부분집합만 허용, tail 단독 사용 금지). **정본: [editorial-fashion.md](editorial-fashion.md) §2 코드블록** — 이 줄은 인라인 복사본이며, `scripts/lint.py`가 정본·이 복사본·검증기 상수를 byte 대조한다.

   | 빼려는 것 | 레인 |
   |---|---|
   | 장면 요소(사람·사물·배경·소품) | **긍정형 재서술** (Tier-0) |
   | 텍스트 렌더 결함(중복·유령 글자·워터마크) | **Tier-1** 화이트리스트 |
   | 정책 안전 단언(화보) | **Tier-2** 페어 |

   `Negative:` 라벨 섹션은 **전 티어 금지**다.
3. **SD-era 폐기 어휘 금지.** `masterpiece/best quality/8k/4k/uhd/trending on artstation/ultra-detailed/highly detailed/sharp focus`. 가중치 `(word:1.3)`, 슬래시 플래그 `--ar/--v`, 본문 `§`, 빈 형용사 `멋지게/감성적으로/고급스럽게/세련되게/beautiful/stunning`도 금지한다. **무대 지정도 동급 노이즈**다. "어워드 수준으로/전문가처럼/최고급"은 기준이 프롬프트 밖에 있어 평균값만 나온다. 수치(여백 %·컬러 60/30/10·위계 단수)·몸 반응·구체 예시로 환원한다.
4. **장비 스펙은 노이즈 → 결과로 환원.** 카메라 EXIF·조명 장비명 대신 결과를 쓴다. 예: `shallow DoF, background falls off softly`, `long soft-edged shadows`, `warm key + cool rim`. 패션의 `Lens character:`·`Director signature:` 라인은 결과+레퍼런스 앵커라 예외다.
5. **수치 명세는 박는다.** HEX 팔레트는 일반 컷당 3~5색, `promo_poster`는 `promo-router.md`의 더 좁은 2~3색 하드 락을 우선한다. 켈빈, `key:fill 1:2` 비율은 품질을 올린다. 숫자가 없으면 평균 이미지가 나오고, 숫자가 있으면 판정 가능한 이미지가 나온다.
6. **1행 = 1컷 = 1호출.** **독립 컷 여러 개**를 한 캔버스에 묶는 그리드/매트릭스 금지. 산출물 자체가 표·패널·슬라이드·도감·만화 페이지면 내부 그리드는 허용 — 1행=1컷=1호출 원칙은 독립 결과물 단위에만 적용. 여러 컷은 N개 별도 행으로 분리한다.
7. **이상적 피부 금지.** 피부는 `natural skin texture, visible pores, subtle film grain`로 쓴다.
8. **실재 상표·실존 인물 참조 금지.** 가상 브랜드/페르소나만 쓴다. 프롬프트 메타에서 모델명·도구명·파일 경로를 말하는 것과 이미지 내용에 실재 상표·인물을 넣는 것은 다른 규칙이다.
9. **생성 후 글자 후처리 절대 금지.** 모든 텍스트는 **프롬프트로 이미지 안에서** 렌더한다. 따옴표 카피 + 롤 라벨 + 자유 작성 존으로 처리한다. 생성된 PNG 위에 코드로 글자를 얹는 합성(PIL/Pillow·ImageMagick·SVG/HTML 오버레이·캔버스 캡처 등)은 금지한다. 폰트·커닝·톤이 원본과 겉돌아 결과물을 망친다. 글자가 틀리면 후처리로 때우지 말고 **프롬프트를 고쳐 재생성**한다. 순서는 타이포 구체화 → `2048x2048` + quality high → 카피 수 축소다.
10. **하중 토큰은 결과와 병기(qualified token).** 필름/룩 이름·조명 패턴·기법(over/underexposure, soft focus, vignetting)·무드 라벨은 **단독 명사로 두지 않는다** — 그 단어가 화면에 만드는 결과를 같은 절에 붙인다. 예: `Kodak Portra 400 film photography, warm creamy skin tones, low contrast`(O) / `Portra 400.`(X). 배치는 조합 공식 `[필름/룩] + [빛의 방향] + [노출 방식] + [색감] + [그레인] + [분위기]`를 따르고, 감성 라벨(웨딩 필름·숲속 무드 등)은 반드시 필름/룩 이름과 병기한다. 스톡·기법·감성별 결과 사전은 [editorial-fashion.md](editorial-fashion.md) §7.4.

### 발전: 철칙 위반 증상 → 원인 역추적 표

| 증상 | 원인 | 고치는 문장 |
|---|---|---|
| 배제한 물체가 오히려 등장 | Tier-0 장면 배제를 `no`로 썼다 | "프레임 안엔 피사체 하나만, 배경은 깨끗한 단색 면" |
| 카피가 두 번 렌더됨 | 같은 따옴표 카피를 본문과 Text-in-image에 중복 배치 | 카피는 롤 라벨 블록 1곳에만 둔다 |
| 앞머리 메타가 이미지 안에 찍힘 | `[AR ...]` 또는 슬롯 토큰이 최종 프롬프트에 잔존 | 메타는 실행 파라미터로 빼고 끝 `AR x:y`만 둔다 |
| 피부가 플라스틱처럼 보임 | 미화 형용사로 피부를 지시 | `natural skin texture, visible pores, subtle film grain` |
| 조명이 장비 카탈로그처럼 산만함 | 장비명·EXIF를 나열 | `warm key + cool rim`, `long soft-edged shadows`처럼 결과를 쓴다 |
| 필름명·감성 라벨을 썼는데 룩 차이가 안 남 | 이름을 단독 명사로 뒀다 — 결과 병기 누락 | `[이름] + [화면 결과]`로 병기: `Gold 200-look, warm golden tones, retro feeling` |
| 복잡한 도표가 빈 카드 4개로 단순화 | 밀도·노드·연결·좌표를 생략 | 섹션 수, 노드 좌표, 화살표 출발→도착, 라벨을 문장으로 고정한다 |

## 3. 마스터 템플릿 — 포맷 A 기본 6섹션 + 끝 AR

순수 서술 + HEX. 첫 섹션이 attention을 가장 강하게 받으니 **핵심 시각정보를 최상단에** 둔다. 헤더 `# 1. Scene` 형식은 허용한다. 본문에 `§`는 쓰지 않는다.

| # | 섹션 | 무엇을 / 분량 |
|---|---|---|
| 1 | **Scene** | 누가·무엇이·어디서·무엇을. 핵심 먼저. 60~120어 |
| 2 | **Camera** | 시점·거리·렌즈 character(결과 서술). 15~30어 |
| 3 | **Lighting** | 방향·soft/hard·그림자·rim light. 장비명 금지. 10~25어 |
| 4 | **Color grading** | 팔레트 + 색온도 + **HEX 3~5개**. 10~20어 |
| 5 | **Texture/Medium** | 매체·질감·표면 반응·후처리. 10~20어 |
| 6 | **Text-in-image** (선택) | `"따옴표 카피"` + 폰트·크기·위치 + legibility 1회. 0~25어 |
| — | 트레일링 | 끝에 `AR x:y` 토큰만 |

## 4. 포맷 A / 포맷 B 라우팅

| 포맷 | 대상 | 형태 | 판정 문장 |
|---|---|---|---|
| **포맷 A — 라벨 6섹션** | 포스터·키아트·인포그래픽·도감·카드뉴스·만화 등 구조물 전반 | 위 표 그대로 | 레이아웃·텍스트·구조가 있으면 A가 맞다. |
| **포맷 B — 화보 플랫 콤마형 단문** | 단독 인물 화보/글래머 에디토리얼 | 라벨 없이 콤마로 잇는 한 문단 | 단독 인물의 표정·피부·포즈·의상이 핵심이면 B가 맞다. |

포맷 B 슬롯 순서는 고정한다. 피사체 → 얼굴 → 헤어 → 장르앵커("한국 남성지풍 클린 화보 컷") → 장면/포즈 → 의상 → 구도 → 조명 → 팔레트 `#HEX`×3~5 → 질감 → [Tier-2 tail] → `AR x:y`. 분량은 **350~450자**, 기본 AR은 `2:3`(1024x1536)이다.

## 5. 사이즈 락 — 러너 프리셋 제약 6종

API는 커스텀이지만 러너 프리셋 경로는 아래 6종으로 잠근다. `auto`는 금지한다. 챕터 안에서는 size를 통일한다.

| ar | size | ar | size |
|---|---|---|---|
| 1:1 | `1024x1024` | 16:9 | `1792x1024` |
| 2:3 / 3:4 / 4:5 | `1024x1536` | 9:16 | `1024x1792` |
| 3:2 / 4:3 | `1536x1024` | 밀집/다컷 | `2048x2048` |

`2:3`만 정확비다. `3:4`·`4:5`는 세로 근사 정규화다. 비지원 값(`1024x1280` 등)은 가까운 6종으로 맞춘다. API 하드 제약은 최대변 3840·16배수·비율 ≤3:1·총픽셀 655,360~8,294,400이다. 투명 배경은 gpt-image-2 미지원(2026-07 실측) — 이 레인의 비목표이며 레인 밖 처리로 표기한다(production.md 모델 팩트).

## 6. 텍스트 렌더 규칙 — 한글 포함

| 규칙 | 작성법 | 판정 문장 |
|---|---|---|
| 정확 카피 고정 | `text reads "오늘 더 따뜻해요"`. 폰트·크기·위치·HEX 지정 | 따옴표 밖에 핵심 카피가 떠돌면 틀렸고, 정확 카피가 따옴표 안에 1회 있으면 맞다. |
| 롤 라벨 분리 | `headline "봄밤 야시장"` / `subhead "4.20 SAT 6PM"` / `callout "선착순 100명"` | 카피 2개 이상이 한 덩어리면 틀렸고, 롤마다 위치·크기·폰트가 있으면 맞다. |
| KO/EN 언어 라벨 분리 | 한 따옴표 문자열 안에 한글+영문 혼합 금지. 렌더 텍스트는 한 줄 한 언어 | 한 줄에 한글+영문이 섞이면 틀렸고, 줄·롤이 분리되면 맞다. |

| 정확도 레버 | **캔버스 크기**. `2048x2048` 실측 12/12 만점. 영문 조어만 ALL CAPS/스펠아웃 최후수단 | 모델 탓으로 끝내면 틀렸고, 캔버스·카피 수·롤 라벨을 조절하면 맞다. |
| 밀집 텍스트 | 카피 블록 3개+ 또는 소형 글자는 `quality: high` + 큰 변 페어링(1536/1792 긴 변 또는 `2048x2048`) | 작은 캔버스에 밀어 넣으면 틀렸고, 긴 변·2048로 글자 높이를 확보하면 맞다. |
| 플레이스홀더 금지 | 렌더 라벨엔 실제 문구만 둔다 | "제목 입력" 같은 대체 문구가 보이면 틀렸고, 최종 카피가 보이면 맞다. |

## 8. 카테고리 라우팅 요약

철칙 전문은 이 파일만 정본이다. C1~C12 컷타입·기본 AR·필수 디테일·공통 DNA·만화 A/B·덱 DNA는 `categories.md`를 본다.

| C1 패션/화보 | C11 시네마틱 키아트 | C3 한국어 포스터 | C4 제품 도감 |
|---|---|---|---|
| **C2 뷰티** | **C5 캠페인** | **C6 인포그래픽** | **C7 카드뉴스** |
| **C8 브랜딩 목업** | **C9 3D 아이콘** | **C10 만화** | **C12 프레젠테이션 덱** |

복잡 도표·도해는 회피하지 말고 정면 시도한다. C6 돌파 전술 + 유형별 레시피 7종(분기 플로우·네트워크·계층·사이클·단면·비교표·데이터 차트)을 적용한다. 연결·노드·수치를 문장으로 구체 지정하고, 수치는 큰 타이포 + 비율감 도형 이중 앵커로 둔다. 안 나오면 단순화 대신 축 변경: 레이아웃 구체화 → 2048 고밀도 → 컷 분할. 고밀도 텍스트 슬라이드는 렌더 한글 **400~800자도 가능**(실측 2026-07, 40컷). 세로 높은 비율(16:9·2:3) + 자유 작성 존이 레버다. 이 밀도에서 소규모 오탈자·간헐 뭉개짐은 정상 산출이므로 치명 카피만 따옴표로 지키고 걸리는 컷만 재생성한다. 후처리 합성은 철칙 9로 금지다. C6·C7은 **밀도가 기본값**이다. 미니멀 플로우·매거진 여백형은 요청이 명시할 때만 쓴다(실측: 나이브 고밀도 컷에 짐).
