# Master Prompt Writer — 디스패치 커널

프롬프트는 지시서가 아니라 **위임 계약**이다. 실행자는 최상위급 추론의 고자율 실행자다 — 방법을 마이크로매니징하지 말고 결과·경계·검증을 계약으로 못 박는다. 이 스킬은 프롬프트 작성의 단일 진입점이며 어떤 에이전트 런타임에서도 동작한다. 런타임별 호출 문법·동반 스킬 매핑은 [references/adapters.md](references/adapters.md).

**references 접근 규칙**: 라우팅·공통 계약만으로 충분하면 references를 읽지 않는다. 모드 골격·채워진 예시·레인 게이트 카드가 필요하면 [references/templates.md](references/templates.md), 모델별 문법이 필요하면 [references/model-playbooks.md](references/model-playbooks.md), 이미지 정밀 컴파일이 필요하면 [references/image/compiler.md](references/image/compiler.md)에서 시작하고, 인물·패션·한국 로컬리티·사진/필름 어휘는 [references/image/editorial-fashion.md](references/image/editorial-fashion.md)까지 읽는다. 이미지 첨부 레퍼런스의 분석 결과를 생성 프롬프트로 컴파일할 때 이 스킬의 레인 게이트가 적용된다.

## 3층 설계 모델

| 층 | 내용 | 실패 시 증상 |
|---|---|---|
| ① 계약층 | 결과 명세·DoD·자율성 3단·하드라인·검증·에스컬레이션 | "다 했다"는데 안 돼 있음, 사고성 행동 |
| ② 구조층 | 산출물 명세(역할·제약·단계·형식+채워진 예시), 난이도 밀도 | 산출물이 얕거나 형식이 매번 다름 |
| ③ 표면층 | 대상 모델/런타임 문법·배치·길이 | 특정 모델에서만 이상 동작, 잘림 |

## 길이 계약 — 2000자 하드라인

1. **붙여넣기 블록당 2000자 이내** (공백 포함). N블록(다중 컷·다중 goal)이면 각 블록이 2000자, N개 메시지로 전달. 내보내기 전 `wc -m` 실측 — 어림짐작 금지, `길이:` 노트에 실측값.
2. **초과 시 오버플로우 문서 — 예외 없는 강제**: 2000자를 넘는 블록을 그대로 내보내는 것은 어떤 이유로도 금지. 감량(4번 순서)으로도 안 줄면 핵심 계약만 프롬프트에 남기고, 긴 명령·상세 지시·자료는 **재부팅 시 비워지는 임시 경로** `/tmp/prompt-handoffs/<slug>.md`에만(휘발성은 의도된 특성 — 프롬프트 잔재를 영구 보관하지 않는다. 워크스페이스 등 영구 경로 금지) **내보내기 전에 실제로 저장**한 뒤 경로 1줄을 첨부한다 — "문서로 빼라"는 지시가 아니라 작성 완료가 게이트다. **전제 조건**: 실행자가 작성자와 같은 파일시스템을 공유할 때만 — 크로스호스트/원격 붙여넣기 대상이면 경로가 실재하지 않으므로 이미지처럼 감량해 자기완결로 만든다. 패딩용 문서 금지.
3. **이미지 계열(IMAGE·COMPOSITE) 예외**: 모델이 파일을 못 읽으므로 경로 참조·오버플로우 금지, 무조건 자기완결. JSON 포맷 요청도 2000자 이내 — 길이 면제 없음. 넘치면 감량 → 컷 분할.
4. 감량 순서: 장식 산문 → 중복 규칙 병합 → how 문장 다이어트("지우면 최상위 실행자가 틀리는가?") → 오버플로우 문서(이미지는 컷 분할).

## 모드 라우팅

| 요청 신호 | 모드 |
|---|---|
| 자율 에이전트 루프에 넣을 지시문 — "goal 써줘", "에이전트한테 시켜줘" | **GOAL** |
| 멀티에이전트/팀 작업, 워커 레인이 이득인 규모 | **TEAM** |
| 단일 작업지시·킥오프·시스템 프롬프트·자동화 잡 | **CONTRACT** |
| 직무 실무 산출물 (수업안·계약검토·원가계산·보고서 등) | **BUSINESS** |
| 특정 모델 문법 튜닝, 리서치/추출 프롬프트, "이 프롬프트 퇴고" | **MODEL** |
| 이미지·영상 생성 프롬프트 | **IMAGE** (하위: gpt-image-2 / Higgsfield / 영상) |
| 원본 사진 피사체 보존 + 배경만 교체 | **IMAGE-COMPOSITE** |
| UI·페이지·컴포넌트를 만들게 할 프롬프트 | 실행 모드 + **DESIGN 오버레이** |
| 이미지 첨부/레퍼런스 분석 결과를 생성 프롬프트로 변환 | **IMAGE** 또는 **IMAGE-COMPOSITE** |
| 직전 산출 프롬프트의 부분 수정 — "톤만 바꿔", "이 줄만 고쳐" | **MODEL-델타** (전체 재작성 금지, 지정 축만) |
| 다단계 프롬프트 체인 — A의 산출이 B의 입력 | 단계별 모드 각각 + **체인 인터페이스 계약** |

Tie-break: ① 자율 루프 지시면 무조건 GOAL. ② 이미지 정밀 컴파일(C1~C12 카테고리·화보 포맷 B·한글 타이포 아트·jsonl 양산)은 내장 컴파일 계층 references/image/ 소관 — 외부 위임 불필요, 최종 산출물은 길이·자기완결 게이트 통과. ③ 영상 프롬프트는 엔진 불문 이 스킬 소관이며(templates의 §영상 공통 규칙은 엔진 불가지), Higgsfield 특화 인테이크·프리셋 라우팅만 higgsfield 브리지 소관이다. ④ 완성 문서 파일(HWP·PDF·덱 조판)은 문서 조판 도구/스킬 소관. ⑤ 슬라이드: 조판이면 조판 도구 / 개별 슬라이드 이미지면 IMAGE(gpt-image-2) / 프롬프트 문법 튜닝이면 MODEL. ⑥ 순수 실행 요청은 해당 실행 경로 소관이다. 단, Higgsfield 프롬프트 작성+생성 혼합 요청은 `higgsfield-prompt-bridge`가 이 스킬의 컴파일 결과를 공식 실행 스킬에 즉시 넘기는 연속 체인으로 처리한다. ⑦ 결과조차 못 정할 만큼 모호하면 요구 구체화 인터뷰 권고 또는 질문 ≤3. ⑧ 혼합 요청은 하나로 합치지 않는다 — 먼저 산출물 유형별 블록으로 분리해 각 블록에 자기 모드·게이트 카드를 적용하고, 분리된 블록 중 한 블록의 산출이 다른 블록의 입력이면 체인 인터페이스 계약(§MODEL)도 추가한다. 분리 사실은 노트에.

모델·브랜드명 두 규칙(혼동 금지): (a) **프롬프트 메타** — 대상 런타임 지정·실존 이미지/영상 엔진명(gpt-image-2·Higgsfield 포함)·경로·명령 허용, 근거 없는 모델 나열 금지. (b) **이미지 콘텐츠** — 생성물 안에 실재 상표·실존 인물 금지(가상 브랜드/페르소나).

## 역할·권한 라우팅

모델명보다 **역할 권한**을 먼저 고른다. 역할·capability·분해·join·blocker·증거 규칙의 정본은 [references/model-playbooks.md](references/model-playbooks.md), 런타임별 매핑은 [references/adapters.md](references/adapters.md)이다. 디스패처는 `prime`/`planner`/`worker`/`critic` 중 필요한 역할만 고르고, 정의를 여기서 반복하지 않는다.

수신자·채널 적응: 기본 전제는 툴 사용 가능한 자율 에이전트 + 메신저 1메시지 전달. 다른 수신자(타 LLM 챗 UI·API 시스템 프롬프트·사람 팀원)나 채널(이메일·문서 삽입) 신호가 있으면 templates §수신자·채널 적응을 적용한다 — 길이 계약은 채널 불문 블록당 2000자 유지, 파일 접근 없는 수신자에겐 경로 참조 대신 인라인.

## 위임 계약 6요소 (계약층 체크 기준)

| 요소 | 계약이 답해야 할 질문 |
|---|---|
| 결과 명세 | 끝난 상태는 어떤 모습인가? (활동이 아니라 상태) |
| Definition of Done | 제3자가 최종 보고만 보고 완료를 판정할 수 있는가? |
| 자율성 3단 | 혼자 결정 / 보고 후 진행 / 절대 금지 — 각각 무엇인가? |
| 하드라인 | 되돌리기 어려운 행동이 이름으로 열거됐는가? |
| 검증 계약 | 완료 보고 전 실행자가 스스로 무엇을 확인하는가? |
| 에스컬레이션 | 막히면 무엇을 시도한 뒤 어떤 형식으로 멈춰 보고하는가? |

자율성 캘리브레이션 — 기본값은 자율 최대: 혼자 결정(되돌릴 수 있고 범위 안: 구현 방식·파일 구조·도구) / 보고 후 진행(범위·비용·인터페이스 변경) / 절대 금지(비가역·외부 접촉 중 **게이트 필요성 테스트 통과분만**). 금지마다 대체 행동(초안·백업 후 dry-run·승인 요청문)을 짝짓는다 — 전면 read-only는 위임이 아니다.

| 모호어 | 치환 방향 |
|---|---|
| 적절히 처리해 | 실패 시 3회 재시도, 그래도 실패면 기록 후 중단 |
| 잘 정리해 | 표 1개 + 5줄 이하 요약 |
| 최대한 자세히/빨리 | 항목당 근거 1개 / 30분 초과 시 부분 보고 |
| 필요하면 X해 | 조건 C가 참이면 X해 |
| 등, 기타, 알아서 | 목록 확정 또는 "이 목록이 전부" 선언 |
| 중요한 것 위주로 | 판단 기준 명시 |
| 완벽하게/깔끔하게/고급스럽게 | 판정 가능한 기준·시각 증거로 환원 |
| nicely / properly / in detail | 영어 동족도 전부 치환 |

## 공통 코어 10규칙

1. **결과 = 상태.** "리팩토링한다" ✗ → "모듈 분리·테스트 전부 통과" ✓.
2. **가짜 완료 테스트.** 가장 게으른 보고("다 했습니다")가 DoD를 통과하면 실패한 DoD — 경로·명령 출력·개수·링크 증거로 통과 불가능해질 때까지 조인다.
3. **자율성 3단 + 하드라인.** "알아서" 위임 금지 — 경계 있는 자율만 안전하다.
4. **검증은 실행자 의무.** 완료 보고 전 스스로 돌릴 절차를 계약 안에. "내가 나중에 확인할게"는 계약 누락.
5. **에스컬레이션.** 막힘 기준·시도 범위·보고 형식(시도/막힌 지점/필요한 것). blocked는 유효 종료.
6. **모호어 0.** 치환표 전수 적용. 제약 모순은 우선순위 1줄로 해소.
7. **컨텍스트 경제.** 장문 자료는 경로 참조(이미지 계열 제외), 인라인 덤프 금지. 답을 바꾸는 사실만.
8. **배치.** 최우선 지시는 앞/뒤, 벌크 자료는 중간, 긴 프롬프트는 자료 뒤 요구 재진술. GOAL은 결과·DoD·게이트 앞배치.
9. **예시 > 형용사.** 형식 요구마다 채워진 예시 1개. 예시-규칙 충돌 시 예시가 이기므로 대조 감사. 부정 지시는 가능하면 긍정 대체.
10. **빈칸은 묻기 전에 채운다.** 추론 우선순위: 대화 컨텍스트 → 저장소/파일 증거(경로가 언급되면 읽어서 확인) → 관례 저위험 기본값(templates §슬롯 자동 채움 표) → 그래도 결과·하드라인이 바뀌는 빈칸만 질문 ≤3. 채운 값은 전부 `가정:` 노트로 투명화 — 사용자가 한 줄로 수정할 수 있게. 컨텍스트 지어내기 금지: 질문이 정당한 건 추론 불가 슬롯(퇴고·델타 기준 원문·합성 원본·정확한 렌더 카피/브랜드명·실계정/실서비스 식별자·비저위험 양자택일 방향 — 정본 목록: templates §슬롯 자동 채움)뿐이다.

## 안전 게이트 카탈로그 (GOAL·CONTRACT·TEAM)

**게이트 필요성 테스트** — 프롬프트에 금지를 넣기 전 3문항이 전부 예일 때만 넣는다: ① 이 행동이 실행자에게 실제로 열려 있는가 ② 실행 환경(승인 훅·권한 설정·시스템 하드라인)이 이미 막고 있지 않은가 ③ 이 작업 범위에서 실제로 발생할 수 있는가. 하나라도 아니오면 그 금지는 노이즈다 — 예: 배포·재시작이 환경 승인 훅으로 이미 게이트되는 실행자에게 "배포 금지"를 다시 쓰는 이중 게이트, GUI 없는 잡에 "스크린샷 유출 금지" 같은 발생 불가 금지.

테스트를 통과한 행동만 게이트 후보 4계열에서 고른다(현재 승인 없으면 검사·준비·초안·승인 요청문까지만): ① 공개·외부 발신(게시·이메일·PR·제3자) ② 자격증명·계정·결제 ③ 비가역 파괴(복구 어려운 삭제·마이그레이션·백업 없는 덮어쓰기) ④ 프로덕션·라이브(실거래·주문·결제 인프라 — 배포·재시작은 대부분의 환경에서 훅이 이미 게이트하므로 테스트 ②에서 걸러진다). 게이트는 좁고 구체적으로 — 허용된 로컬 작업엔 차단 대신 백업·dry-run·diff 요구. 비밀값 노출 요구 금지. "범위 밖" 선언(예: push는 이 작업 범위 밖)은 게이트가 아니라 범위 정의다 — 금지 어휘 대신 범위 문장으로 쓴다.

## 모드별 요점 (상세·골격·예시는 전부 templates.md)

- **GOAL**: 자율 루프의 판정자는 지시문 앞부분+최종 보고만 본다고 가정 → 핵심 앞배치, 런타임 턴 예산 내 완료/blocked 스코프, 최종 보고에 변경/검증/미수행+사유 요구. 분할 규칙: §GOAL.
- **TEAM**: 목표/범위(+비목표)/컨텍스트/작업 방식(레인)/검증/산출물/경계/상세자료 필드. 명령형. 모호하면 요구 구체화 인터뷰 권고. §TEAM.
- **CONTRACT**: 6요소 골격. 자동화 잡=무인 변환("보고 후 진행"→"건너뛰고 기록")+멱등성, 시스템 프롬프트=상시 규범 변환. §CONTRACT.
- **BUSINESS**: 직무 5요소(역할+전문성/현실 제약/목표 2+/단계/형식+예시) + 난이도 1~7 밀도 + 직업 치환. §BUSINESS.
- **MODEL**: 플레이북 적용(model-playbooks.md) — 역할·권한 먼저, 실제 모델명은 호환 노트나 런타임 어댑터에서만. 단계 강제 금지, 모델 능력 지어내기 금지, 필요한 권한보다 비싼 역할 고정 금지, 주장 스탬프 만료 시 재검증. 퇴고=3렌즈 후 수정본+변경 노트만. 델타 수정(전체 재작성 금지·지정 축만, 기준 원문 없으면 추론 불가 → 요청)·리서치/추출 골격·프롬프트 체인(인터페이스 계약): §MODEL.
- **IMAGE**: 자기완결 2000자. 엔진 라우팅 먼저(텍스트 렌더=gpt-image-2 / 인물 리얼리즘·동일 인물·카메라 모션=Higgsfield / 피사체 보존 배경 교체=COMPOSITE) → 해당 레인 게이트 카드 적용. §IMAGE. 정밀 컴파일은 내장 계층 [references/image/compiler.md](references/image/compiler.md)부터 진입(카테고리·패션/화보·한국 로컬리티·룩/컨셉·타이포·양산 6파일 + `scripts/check_prompt.mjs` 검증기). 한국인/한국 공간은 고정관념이 아니라 관찰 가능한 스타일·공간·조명·잡지 톤으로 쓴다. **Soul HEX는 예외 계약**이다: Soul 2.0의 실제 제어는 웹 `Color signature`에 레퍼런스 1~20장을 넣어 전체 팔레트·무드·컬러 그레이드를 가져오는 방식이며, 수동 대상별 HEX 3~5개를 Soul 프롬프트에 강제하지 않는다. 추출 HEX는 타 모델/QC로 넘길 때만 출처가 붙은 `observed_palette` soft metadata로 쓴다. 정확한 의상·제품·브랜드색은 별도 스와치/레퍼런스와 `critical_colors` 계약으로 분리한다. Higgsfield 실행 동사가 함께 있으면 `higgsfield-prompt-bridge`의 연속 체인 계약으로 넘겨 같은 요청 안에서 생성·결과 검증까지 끝낸다.
- **IMAGE-COMPOSITE**: 작업 재정의 선언, 연산 금지 계약은 부정형 정당(장면은 긍정형), 재조명 다이얼 1단 필수, 판정 문장, 실패 조건+FAIL 지시. §배경 합성 레인.
- **DESIGN 오버레이**: "기본 프롬프트로도 나왔을 결과라면 부족하다" — 방향 잠금(이름 붙은 미학+시그니처 1개), 레퍼런스 3속성 분해, 시각 테제 3줄, AI 슬롭 금지 계약, 뷰포트 검증. GOAL 모드에선 선제출→최종 보고 자가 검증으로 변환. §DESIGN.

## 안티패턴

단계 마이크로매니징(→ 결과·검증만) / 장문 인라인 덤프(→ 경로 참조) / 전면 read-only 과잉 차단(→ 좁은 게이트+대체 행동) / **시스템 이중 게이트**(환경 훅이 이미 막는 배포·재시작류를 프롬프트가 재금지 → 필요성 테스트로 제거) / **질문 반사**(빈칸을 이유로 질문부터 던짐 → 추론·기본값으로 채우고 `가정:` 기록, 질문은 추론 불가 슬롯만) / "알아서" 위임(→ 3단 경계) / 하나의 비대 goal(→ 분할) / 예시-규칙 충돌 방치(→ 대조 감사) / 무대 지정 어휘("어워드 수준으로" → 수치·구체 예시) / 패딩 문서. 이미지 계열 안티패턴은 레인 게이트 카드 소관.

## 워크플로우

1. **분류** — 모드+레인 선택(Tie-break 적용). 빈 슬롯은 §슬롯 자동 채움 우선순위로 채우고 `가정:`에 기록 — 질문은 추론 불가 슬롯만 ≤3.
2. **결과 명세** — 끝난 상태 1~2문장, 자료 인벤토리(로컬은 읽고 URL은 추출, 사실 지어내기 금지).
3. **계약층** — DoD(가짜 완료 테스트로 조임)→자율성→하드라인→검증→에스컬레이션+안전 게이트.
4. **구조층** — 모드 골격(templates.md)에 채우기, 형식엔 채워진 예시 1개.
5. **표면층** — 대상 모델 문법(model-playbooks.md), 전문성은 구체로 프라이밍, 역할 없는 단어 삭제.
6. **감량 패스** — 모호어 스캔, 모순 확인, how 다이어트.
7. **길이 게이트** — 블록당 `wc -m` 실측. 초과 블록은 그대로 내보내지 않는다: 감량 → 그래도 초과면 오버플로우 문서를 실제로 작성해 경로 1줄로 대체(이미지는 컷 분할).
8. **드라이런+린트** — 현실 입력 2~3개(엣지 1개)로 멘탈 드라이런, 드러난 구멍 수정. 레인 게이트 카드 대조 후 출력.

## 보강 루프 — 산출 후 후속 요청 재진입

전달한 프롬프트에 오는 후속 요청은 새 작업이 아니라 **워크플로우 재진입**이다. 유형을 분류해 최소 진입점부터 다시 타고, **출력 게이트를 신규 작성과 동일하게 다시 통과**한다. 상세·진단표: [references/templates.md](references/templates.md) §보강 루프.

| 후속 유형 | 진입점 |
|---|---|
| (a) 스코프 추가(요구 증가) | 같은 결과면 완료 기준·게이트에 패치, 독립 결과면 별도 블록/goal (Tie-break ⑦·GOAL §스코프 분할 규칙) |
| (b) 방향 수정(계약 일부 뒤집힘) | 단일 축이면 §MODEL 델타 수정, 결과 명세·모드·레인이 바뀌면 워크플로우 1단(분류)부터 재작성 |
| (c) 압축/감량 | §길이 계약 감량 순서 재적용 후 재실측 — 계약 요소는 감량 대상 아님 |
| (d) 실행 실패 환류 | 증상→결함 계약 요소 진단 후 그 요소만 조임 (진단표: templates §보강 루프) |
| (e) 모호("좀 더 좋게") | 모호어 — 3층 설계 모델 표에서 최약 축을 추론해 `가정:`으로 처리, 축이 갈릴 때만 질문 ≤1 |

**패치 vs 재작성**: 결과 명세·모드·레인이 보존이면 패치, 하나라도 깨지거나 계약 요소 3개+를 손대야 하면 재작성(패치가 남길 모순 감사보다 재작성이 싸다). **재검증 불변식**: 보강 후에도 블록당 2000자 실측·이미지 자기완결·레인 게이트 카드·게이트 필요성 테스트를 다시 통과 — 패치로 2000자 초과 시 감량→오버플로우 문서(이미지 컷 분할). **체인 무결성**: 산출 스키마(인터페이스)를 건드리는 보강은 §프롬프트 체인 정합 재증명(다운스트림 동반 수정), 내부 보강은 단독 패치.
**자동개선 옵션 추천(조건부)**: 산출 직후 자가 진단해 **실제 약한 축이 잡힐 때만** 진단 1줄+옵션 1줄을 덧붙이고, 수락 시 위 표 (b)/(c)/(e)로 흘려보낸다(방향 수정은 소관 밖). 형식적 상시 추천 금지. 문안: templates §보강 루프.

## 출력 게이트 — 전부 통과 시에만 내보낸다

- [ ] 결과가 활동이 아니라 상태인가
- [ ] "다 했습니다" 한 줄이 DoD를 통과하지 못하는가
- [ ] 6요소가 있는가 (GOAL·TEAM·CONTRACT 전부 / BUSINESS·MODEL·IMAGE 계열은 결과·형식·검증 3요소만 필수)
- [ ] 모호어 0, 예시-규칙 충돌 0, 제약 모순 0
- [ ] 프롬프트 게이트마다 필요성 테스트(열려 있음·시스템 미차단·발생 가능) 통과 — 이중 게이트 0
- [ ] 형식 요구마다 채워진 예시 1개
- [ ] **블록당 2000자 실측 — 초과 블록 0.** 초과분은 내보내기 전에 실제 작성된 오버플로우 문서 경로(동일 FS일 때만)로 대체됐는가, 크로스호스트면 자기완결인가
- [ ] **멘탈 드라이런을 실제로 수행하고 드러난 구멍을 수정했는가**
- [ ] 빈 슬롯이 질문 대신 추론·기본값으로 채워졌고 `가정:`에 기록됐는가 (질문은 추론 불가 슬롯만)
- [ ] IMAGE 계열·DESIGN: **해당 레인 게이트 카드 통과** ([references/templates.md](references/templates.md) §레인 게이트 카드)
- [ ] GOAL: 결과·DoD·게이트 앞배치 + 턴 예산 현실적(아니면 분리 제안)
- [ ] 언어 매칭(한국어 요청→한국어, 경로·명령·심볼·제품명 원문 유지)
- [ ] 남은 how 문장마다 존재 이유가 있는가

## Output format

완성 프롬프트를 복붙 가능한 코드블록으로 출력(블록당 2000자, 다중이면 블록 분리+노트에 명시). IMAGE·COMPOSITE 단일 편집 프롬프트 통합 규칙은 [references/templates.md](references/templates.md) §레인 게이트 카드가 정본이다. 코드블록 뒤에는 필요할 때만: `가정:` `필수 질문:`(≤3) `분리 제안:` `상세자료:`(실존 경로) `길이: N자`(실측). 인사·과정 설명·장식·가짜 옵션 메뉴 금지 — **프롬프트가 제품이다.**
